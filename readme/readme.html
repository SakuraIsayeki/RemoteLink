Visual C++ pour le développement sous Linux

Configuration de votre projet pour le développement Linux

Avec cette charge de travail, vous pouvez créer du code C++ pour les serveurs, les postes de travail et les appareils Linux. Vous pouvez gérer vos connexions à ces machines à partir de Visual Studio. Visual Studio copie et génère à distance vos sources automatiquement. Il peut lancer votre application avec le débogueur. Notre système de projet prend en charge le ciblage d'architectures spécifiques, notamment ARM.

Connexion à Linux

Prérequis

Nous prenons uniquement en charge la génération à distance sur la machine cible Linux. Nous ne sommes pas limités par des distributions Linux spécifiques, mais nous avons des dépendances quant à la présence de certains outils. Plus précisément, nous avons besoin de openssh-server, g++, gdb et gdbserver. Utilisez votre gestionnaire de package favori pour les installer, par exemple sur les systèmes basés sur Debian : sudo apt-get install openssh-server g++ gdb gdbserver

Première connexion

La première fois que vous ciblez une machine Linux, vous êtes invité à entrer des informations de connexion. Cela est déclenché par la génération du projet.

Ajout et suppression de connexions

Pour ajouter une nouvelle connexion, accédez à Outils > Options, puis recherchez Connexion. Le Gestionnaire des connexions se trouve sous Multiplateforme. À partir de cet emplacement, vous pouvez ajouter et supprimer des connexions.

Pour changer la connexion utilisée par un projet, accédez aux paramètres généraux des propriétés du projet, puis mettez à jour l'option Machine de build distante.

Propriétés du projet

Toutes les options nécessaires pour contrôler la compilation C++ sont exposées sur les pages de propriétés du projet. Nous allons aborder certains aspects du fonctionnement de Linux. Tout d'abord, sous les paramètres généraux, vous allez voir que la racine distante a la valeur ~/projects/ par défaut, et que nous définissons le répertoire du projet distant pour qu'il corresponde au nom de notre projet à cet emplacement.

En regardant les paramètres généraux du projet, vous pouvez voir de quelle façon nos répertoires de sortie et intermédiaires ont été configurés. De plus, vous allez voir que ce projet a été configuré en tant qu'application. Ainsi, notre exécutable se trouve sous bin/x64/Debug/ en tant que ConsoleApplication1.out. Notez que pour les types de configuration, nous prenons également en charge les bibliothèques statiques et dynamiques.

Ajoutez des dépendances de bibliothèque supplémentaires dans la page de propriétés Éditeur de liens> Entrée.

Vous pouvez passer des commandes de prélancement supplémentaires au débogueur pour effectuer des actions telles que lancer des applications graphiques sur la machine Linux distante.

Vous pouvez également envoyer des événements postbuild pour contrôler le comportement à distance, comme dans cet exemple qui exporte une broche gpio à utiliser sans nécessiter l'exécution de l'exécutable en tant que super utilisateur.
